## Guidelines

### Message Structure
- **Start with checkmark** âœ… to indicate completion
- **Use emojis** for visual organization (ğŸ¯, ğŸ“, ğŸ“š, ğŸ”—)
- **Keep bullet points** concise and user-focused
- **Include clear navigation** path for accessing the feature
- **Provide both user and technical** references

### Content Tips
- **Focus on user benefits** rather than technical implementation
- **Use simple language** that non-technical staff can understand
- **Include item types** relevant to the specific feature (medicines/store items/lab items)
- **Mention export capabilities** when available
- **Reference all procurement types** for comprehensive coverage

### Link Format
- **Wiki links** should use the actual GitHub wiki URL format
- **Issue links** should reference the original GitHub issue number
- **PR links** should reference the implementation pull request

### Audience Considerations
- **Hospital administrators** need to understand business value
- **End users** need clear access instructions
- **IT teams** need technical reference links
- **Training coordinators** need user guide references

## When to Use

- **Feature completion** announcements to hospital groups
- **System updates** that affect user workflows  
- **New functionality** rollouts requiring staff training
- **Enhancement notifications** for existing features

## Distribution

- **WhatsApp groups** for hospital stakeholders
- **Email notifications** for formal communications
- **Slack channels** for development team updates
- **Training materials** as reference format
